或许能用 ChatGPT 插件实现财富自由

我们知道，当前 ChatGPT 最大的局限性就是模型训练数据是其唯一的信息来源，它的信息也许已经过时，解决办法或许不再适用当前的程序，因为 ChatGPT 学习到的 “知识” 仅止步于 2021 年 9 月。

此外，ChatGPT 仅提供文本输出， 如果想要其它的形式，就得对其结果进行额外处理。

当然，4.0 将支持其它形式，如图片

那有什么办法可以解决这一个问题呢？我们又能从中探索出什么机会呢？

好吧，答案就是：ChatGPT 插件

文章标题说是，或许能用 ChatGPT 插件实现财富自由，肯定很想知道答案吧，我们这就一步一步来为你解答。

1. 认识一下

1.1 是什么

我们把 ChatGPT 想象成一名 ”顶级的语言学家“，它学习了互联网上截止 2021 年 9 月的所有语言知识。它能理解自然语言，能够根据所学知识做出回答，但它还有以下这些不足：

  1. ”知识“止步于 2021 年 9 月

  2. 虽然语言好，也有学习各行各业的知识，但数学、物理、生物化学这些特定领域的东西，对它来说还是有点难，能做一些解释，但不是很好，很准确

  3. 它只能通过写文字来回答，还不能通过画图、朗读等方式回答

    这位 ”顶级的语言学家“ 只有一双手，没有嘴巴，没有眼睛，这双手还只会写字。

  4. 有些具体的事情它也做不了，比如订餐、定航班、买东西购物

那怎么解决这些不足呢？这不就需要插件了哇！

那插件是什么呢？

这里用最简单的方式来回答：ChatGPT 这名 ”顶级的语言学家“ 自身能力不足，于是就寻求第三方能力来帮它，可以提供这些第三方能力的，就是 ChatGPT 插件。

1.2 怎么用

非常的简单，在插件市场选择安装，然后在 ChatGPT 就会充当一个智能 API 调用者的角色，ChatGPT 会主动调用所匹配的 API 来执行操作。例如，如果用户问：“我想在深圳住几晚”，模型可能会选择调用酒店预订的插件 API，然后接收 API 响应，最后结合响应数据生成面向用户的更合理更丰富的答案。

具体 ChatGPT会去调取哪一个插件目前还没有详细的说明。

3. 怎么做到的

如果你对这部分不感兴趣，可以直接略过，这里主要讲 ChatGPT 是怎么与插件交互，最终得到期望的结果的。

这里以 Wolfram 插件为例来说明。

这是怎么做到的？我们可以从 Used Wolfram 框的内容，如下图，得出以下结论：

  1. ChatGPT 理解用户的输入，得到一个具体的查询

    How far is it from Chicogo to Tokyo ——> distance from Chicago to Tokyo

  2. ChatGPT 向 Wolfram 发起一个请求，并将理解后的结果作为参数

  3. Wolfram 根据请求返回一个详细的结果，知识型的结果

  4. ChatGPT 获取到 Wolfram 的结果，并进行丰富、转译、修饰等

    Wolfram 返回了 aircraft (550mph) | 11 hours 30 minutes

    ChatGPT 将其丰富为 If you were to travel by … it would take about …

只要是使用的插件，都可以通过点击插件对话框来看到具体的请求和返回内容。

4. 财富自由

讲了半天插件，那这个插件怎么才能让我财富自由呢？

相信我，觉得不是标题党。

ChatGPT 插件一出，这就是一个绝佳的机会？为什么这么说：

  1. 可以认为 ChatGPT 的所有用户都是你插件的潜在用户
  2. 只要插件形成习惯，那么这些用户留存率极高，同时传播极快且无成本

只要插件有了用户，那么在插件中赚钱岂不就是流量变现了。

我来举几个自己想到的例子哈！

4.1 生活类插件

  • 人是离不开衣食住行的，比如：机票酒店订购、旅游安排等，利用插件就可以将这些与 ChatGPT 结合起来，将用户引流到指定的平台，那这个插件还不赚翻了，甚至，自己也可以做生活平台哟！

  • 网上购物同样是一个可造之地，同样的方式，插件引流，而且，插件可以从多个购物平台中获取多种购买方案供使用者参考，只要使用者通过插件引流购买的，你不就赚了。

4.2 品牌推广类

  • 很多时候，我们会在小红书等平台插件某某商品是否值得购买、某某电器应该买哪一个牌子的、某某游戏玩起来如何等等，用一个插件专门来实现这些，如品牌推广、种草以及宣传等，你说你能不搞钱？
  • 除了实体商品外，还有很多也是需要推广的，比如教育培训、歌曲、电视剧等，任选一款，做大做好。

5. 限制

  1. OpenAI 大佬自己会做

    上面谈到的这些类插件，OpenAI 自己就会去实现的，而且他们拥有着天然的优势。

    但反过来一想，虽然他们也回去实现这些个插件，但它并不能覆盖所有的场景，所以还是有希望的。

  2. 插件有了，怎么变现

    这其实是个商业问题了，产品有了，怎么引流，怎么变现，甚至别的有实力（资本）的团队还可以模仿你的插件，然后自己用钱砸出流量来。

    似乎又回到了起点，没有资本就很难引流，没有流量就很难变现，狗头。

但，总的来说，我觉得这都是一个很好的机会。感兴趣的同学可以等 OpenAI 开放了后去做一两个插件感受一下。
