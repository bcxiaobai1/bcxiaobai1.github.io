微软扩大与Meta的AI合作，强强联合，友商岌岌可危？

微软正在继续扩展其在Azure上的AI服务的功能，Meta正在注意到这一点。

微软和Meta正在扩大他们正在进行的AI合作伙伴关系，Meta选择Azure作为“战略云提供商”，以加速自己的AI研究和开发。微软官员在Microsoft Build 2022开发者大会的第二天分享了有关Microsoft-Meta合作伙伴关系的最新细节。

微软和Meta在2017年宣布了ONNX（开放神经网络交换）格式，使开发人员能够在不同的AI框架之间移动深度学习模型。微软于2018年开源了ONNX Runtime，这是ONNX格式模型的推理引擎。

今天，Meta官员表示，他们将使用Azure来加速Meta AI集团的研发。Meta 播放使用 5400 个 GPU 的专用 Azure 群集，使用 Azure 中的最新虚拟机 （VM） 系列（NDm A100 v4 系列，具有
NVIDIA A100 Tensor Core 80GB GPU）来执行其一些大型 AI 研究工作负载。去年，Meta开始使用Azure VM进行一些大规模的AI研究，特别是分布式AI训练。

两人今天还表示，他们将合作在Azure上扩展PyTorch的采用。去年，微软公开承诺加强对在Azure上使用Meta的PyTorch深度学习框架的企业客户的支持。我认为这里的新部分是微软承诺“在未来几个月内”构建新的PyTorch开发加速器，以促进在Azure上快速实施基于PyTorch的解决方案。

人工智能开发一直是本周Build的一大主题，微软宣布在高通骁龙平台上运行一个新的基于Arm的开发套件，该套件将于今年晚些时候推出。即将到来的“Project Volterra”开发套件的最大声望之一是它能够在Arm上运行AI工作负载，比其他处理器更快，更高效。（微软仍然没有发布任何技术规格，包括有关将搭载的处理器的信息。

Microsoft 新宣布的 Power Apps Express Design 功能使用 Azure Cognitive Services 中的 AI 模型将绘图、图像、PDF 和 Figma 设计文件转换为应用程序。

微软本周还宣布，Azure OpenAI服务现已推出预览版。预览版中的那些人将能够从OpenAI访问不同的模型，包括GPT-3基本系列（Ada，Babbage，Curie和DaVinci），Codex系列和嵌入模型，并将它们与Azure的企业功能相结合。2019年，微软在OpenAI上投资了10亿美元，以换取OpenAI的承诺，使微软成为“将新AI技术商业化的首选合作伙伴”。

另一个 Microsoft 认知服务，即语言认知服务，现在提供文档和对话的摘要。微软官员建议，这可能对呼叫中心的使用有用。Microsoft 正在正式发布新的语言服务功能，包括自定义命名的实体识别，这可以帮助开发人员使用客户的域特定标签（如支持票证或发票）对文本进行组织和分类。

微软还宣布Azure Machine Learning Responsible AI仪表板处于预览状态。微软官员表示，这种能力将帮助开发人员和数据科学家更轻松地实施负责任的人工智能。

微软目前的AI模型集包括用于丰富语言理解的图灵，用于语言翻译的Z-Code和用于视觉识别的Florence。其 Azure 认知服务系列包括视觉、语音、语言、决策的构建基块服务，以及其 OpenAI 服务。现有的 Azure 应用 AI 服务包括认知搜索、表单识别器、沉浸式阅读器、机器人服务和视频分析器。
