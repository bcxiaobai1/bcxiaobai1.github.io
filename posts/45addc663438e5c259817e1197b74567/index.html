C/S和B/S架构介绍

1.系统架构包括哪些形式？

  • C/S架构
  • B/S架构

2.什么是C/S架构？

  • 说白了就是客户端/服务端，我们需要安装特定的客户端软卷，例如：QQ。

  • C/S架构的优缺点是什么?

    • 优点：
      • 速度快（软件中的数据大部分都是集成到客户端软件当中的，很少量的数据从服务器端传送过来，所以C/S结构的系统速度快）
      • 用户体验好（速度又快，页面又酷炫（相较于页面单调的浏览器），当然体验好了）
      • 界面酷炫（专门的语言去实现界面的，更加灵活。）
      • 服务器的压力小（因为大量的数据都是集成在客户端软件中，所以服务器只需要传送很少的数据量，当然服务器压力小）
      • 安全（因为大量的数据都是集成在客户端软件中，并且客户端有很多个，服务器虽然只有一个，就算服务器那边出现问题（地震，火宅，受损），问题也不大，因为大量的数据在客户端软件上有缓存）
      • …
    • 缺点：
      • 升级维护麻烦（成本比较高，每一个客户端软件都需要升级，比如：银行的系统升级，需要专业人士去一个个升级，有一些软件不是那么容易升级安装的）

3.什么是B/S架构？

  • 说白了就是浏览器/服务器，实际上B/S架构就是一个特殊的C/S架构，只不过这个C比较特殊，是浏览器软件

  • B/S架构的优缺点：

    • 优点：
      • 升级维护方便，成本比较低。（只需要升级服务端即可）
      • 不需要安装特定的客户端软件，只需要打开浏览器，输入网址。
    • 缺点：
      • 速度慢（不是因为带宽低的问题，是因为所有的数据都是在服务器上的，用户发送的每一个请求，都是需要服务器全身心的响应数据的，所以B/S架构的系统在网络中的数据传输量比较大。）
      • 体验差，页面不酷炫（因为浏览器只支持三个语言HTML CSS JavaScript，而且速度慢）
      • 不安全（所有的数据都存储在服务器上，只要服务器受损，数据全部丢失）
      • …

4.B/S和C/S架构哪个好，哪个不好？

  • 这个问题问的很没有水平，并不是哪个好，哪个不好。不同的结构的系统在不同的业务情景下有不用的适用场景。
  • 大型游戏等娱乐性软件建议使用什么？
    • 建议使用C/S架构
  • 公司内部使用的一些业务软件推荐使用什么？
    • B/S架构
      • 因为公司内部使用的系统，需要维护成本低。
      • 因为公司内部使用的系统，不需要酷炫的页面。
      • 因为公司内部使用的系统，主要的功能是进行数据的维护即可。

5.开发B/S架构的系统，其实就是在开发网站，其实就是在开发一个web系统。

  • 开发一个web系统，需要哪些技术？
    • web前端（运行在浏览器上的程序）
      • HTML
      • CSS
      • JavaScript
    • web后端（运行在web服务器上的程序）
      • Java可以（Java做web开发，我们称为JavaWeb开发。JavaWeb开发最为核心的规范：Servlet规范【说白了就是运行在服务器端的Java小程序】）
      • c语言也可以
      • c ++ 也可以
      • Python也可以
      • PHP也可以
      • …

6.JavaEE是什么？

  • Java分为三个部分

    • JavaSE
      • Java标准版（一套类库：别人写好的一套类库，只不过这个类库是标准类库，不管以后走EE，或者走ME，SE都是基础，必须先学）
    • JavaEE
      • Java企业版（也是一套类库：也是别人写好的一套类库，只不过这套类库可以帮助我们完成企业级项目的开发，专门为企业内部提供解决方案的一套（多套）类库）
      • 可以用来开发web系统
      • Java比较火爆的就是EE方向
    • JavaME
      • Java微型版（还是一套类库：只不过这套类库是用来帮助我们进行电子微型设备内部程序的开发）
      • 例如：机顶盒内核程序，吸尘器内核程序，电冰箱内核程序，电饭煲内核程序…
    • JavaEE实际上包含很多种规范：13中规范。其中Servlet规范就是JavaEE规范之一，学习Servlet规范，还是Java语言。
