vue前后端部署,域名解析、服务器等配置

1、申请域名、服务器；

域名、服务器可以再阿里云或者华为云购买即可；域名需要解析，记录值为服务器的公网ip。

2、安装Xshell 、Xftp ;

安装过程需要服务器用户名、密码 ,主机名为公网ip，用户名默认root 。

3、在Xshell里安装nginx;

        1.因为Nginx依赖于gcc的编译环境，所以，需要安装编译环境来使Nginx能够编译起来。

命令：

yum install gcc-c++

        2.Nginx的http模块需要使用pcre来解析正则表达式，需要安装pcre。

命令：

yum install -y pcre pcre-devel

        3.安装依赖的解压包。

命令：

yum install -y zlib zlib-devel

        4.ssl 功能需要 openssl 库，安装 openssl。

命令：

yum install -y openssl openssl-devel

        5.下载Nginx,进入Nginx官网，

Nginx官方提供了三个类型的版本：

Mainline version：Mainline 是 Nginx 目前主力在做的版本，可以说是开发版

Stable version：最新稳定版，生产环境上建议使用的版本

Legacy versions：遗留的老版本的稳定版我们选择Stable version，点击下载。

下载完成后解压，将解压的nginx通过Xftp移动到 右边的 /usr/local/  如nginx名字为nginx-1.20.1（注：不要重命名）

        6.安装完后回到Xshell 依次执行以下命令；

cd /usr/local/nginx-1.20.1

./configure --prefix=/usr/local/nginx

sudo make

sudo make install

如果第二部出现权限问题，则在Xftp右边的nginx-1.20.1 里找到configure文件，右键“更改权限” 右上角全部更改为 777 。

        7.配置nginx.conf；

输入命令：

vim /usr/local/nginx/conf/nginx.conf 

默认端口是80，因为可能会和Apache使用的端口冲突，这里改为8089；server_name修改为公网IP地址就可以 （输入i变成可编辑状态，修改完后保存退出 先esc按键，然后 :wq  进行保存）。

        8.启动Nginx；

进入 /usr/local/nginx/sbin 目录，执行命令：

./nginx 

然后查看Nginx进程是否启动：

        9.浏览器输入公网ip+端口号，看能否显示nginx。

4、安装java JDK 

yum list installed | grep

查看JDK软件包列表

yum search java | grep -i --color jdk
yum install -y java-1.8.0-openjdk*

配置环境变量

JDK默认安装路径/usr/lib/jvm　　

注意修改 java-1.8.0-openjdk-1.8.0.181-3.b13.el7_5.x86_64目录

命令：vim /etc/profil  然后再下面插入以下代码，先按 i 进入可编辑状态然后复制以下代码放在最底下

export JAVA_HOME=/usr/lib/jvm/jre-1.8.0-openjdk-1.8.0.292.b10-0.el8_3.x86_64/

export JRE_HOME=$JAVA_HOME/jre

export CLASSPATH=$JAVA_HOME/lib:$JRE_HOME/lib:$CLASSPATH

export PATH=$JAVA_HOME/bin:$JRE_HOME/bin:$PATH

 

然后保存关闭文件先按esc 退出键，然后命令为 :wq，执行如下命令生效

source  /etc/profile
  • 使用如下命令，查看JDK变量 
echo $JAVA_HOME
echo $PATH
echo $CLASSPATH

 5、把我们的前端代码dist包、后端jar包 通过Xftp放在目录 /home 下

接着通过Xshell 配置我们项目的端口号和ip;

输入命令：

vim /usr/local/nginx/conf/nginx.conf 

接着输入i 进入可编辑状态 然后辅助一个server ，里面有localtion,如图

最后按 esc按键 ,然后命令  :wq  保存。

6、最后两步是启动nginx 和 jar 包

        1、进入nginx 目录下的sbin目录 输入载入命令：

./nginx -s reload 

        （如果后期只要是修改了nginx 的config文件，都需要重新载入./nginx -s reload）

        2、进入存放jar 包的目录启动jar包 ，如home目录下输入命令 ：

nohup java -jar redisson-demo-0.0.1-SNAPSHOT.jar &

        3、完成，如果你是ip，则浏览器输入ip访问，如果是域名，则域名访问。
