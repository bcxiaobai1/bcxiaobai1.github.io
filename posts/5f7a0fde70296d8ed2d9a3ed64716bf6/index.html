Anaconda镜像源操作指令集

Anaconda镜像源操作指令集

Anaconda中，安装相关库包时，需要添加合适的镜像源channels。合适的镜像源提供相关库包集，添加镜像源可以极大地提高下载网速。

# 在命令行中，激活编译环境
conda activate neroic

1.查看镜像源

1.1 show指令

conda config --show channels，查看所有已添加的镜像源。

conda config --show channels

channels:

  • https://mirrors.bfsu.edu.cn/anaconda/cloud/bioconda/
  • https://mirrors.bfsu.edu.cn/anaconda/cloud/conda-forge/
  • https://mirrors.bfsu.edu.cn/anaconda/pkgs/free/
  • …(省略多项)
  • https://mirrors.ustc.edu.cn/anaconda/pkgs/free/
  • defaults
    其中，defualts为Anaconda自带的镜像源网址anaconda.org。

1.2 安装错误时的提示

如果记不住前述的show指令，可以随意安装一个明显不存在的库包。
安装无法搜索到的库包时，会列出现有的镜像源Current channels。

conda install xiaoyuzhou

Current channels:

  • https://conda.anaconda.org/pytorch/win-64

  • https://conda.anaconda.org/pytorch/noarch

  • https://conda.anaconda.org/nvidia/win-64

  • https://conda.anaconda.org/nvidia/noarch

  • https://mirrors.bfsu.edu.cn/anaconda/cloud/bioconda/win-64

  • https://mirrors.bfsu.edu.cn/anaconda/cloud/bioconda/noarch

  • https://mirrors.bfsu.edu.cn/anaconda/cloud/conda-forge/win-64

  • https://mirrors.bfsu.edu.cn/anaconda/cloud/conda-forge/noarch

  • …(省略多项)

  • https://repo.anaconda.com/pkgs/msys2/win-64

  • https://repo.anaconda.com/pkgs/msys2/noarch

1.3 直接访问镜像源网址

分别打开三个镜像源网址，对比分析。

https://conda.anaconda.org/nvidia/
https://conda.anaconda.org/nvidia/win-64
https://conda.anaconda.org/nvidia/noarch

打开镜像源网址https://conda.anaconda.org/nvidia/，可看到包含带有多类操作系统标签的文件夹链接。
noarch是no architecture的缩写，说明这个镜像源网址下的包可以在各个不同的CPU上使用。

打开镜像源网址：https://conda.anaconda.org/nvidia/win-64，可以看到文件夹nvidia/win-64下的文件列表，包含不同nvidia版本的文件包，且文件包命名一般均以cuda开头。

而打开镜像源网址https://conda.anaconda.org/nvidia/noarch，可以看到文件夹nvidia/noarch下的文件列表，该网址下包含的是直接在CPU下使用的库包。

2.添加镜像源

conda config --add channels https://…，添加镜像源。

conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/

3.指定镜像源进行安装

conda install numpy -i https://…，安装库包，可以指定镜像源，从该镜像源中搜索下载。

pip install numpy -i https://pypi.tuna.tsinghua.edu.cn/simple

4.删除指定的镜像源

conda config --remove channels https://…，删除指定的镜像源

conda config --remove channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/

5.删除所有镜像源

conda config --remove-key channels，删除所有镜像源

conda config --remove-key channels

6.设置搜索时显示通道地址

conda config --set show_channel_urls yes，设置搜索时显示通道地址

conda config --set show_channel_urls yes

7.设置镜像源灵活优先权

conda config --set channel_priority flexible，设置镜像源灵活优先权

理论上，添加更全的镜像源，可以保证更全相关库包的搜索与下载渠道。但若将conda install …指令改成pip install ...时，由于过多的镜像源，系统可能会显示警告并提示镜像路径过多。且包含有不同的镜像源，在未设置灵活的搜索顺序时，系统默认会按字母排列顺序进行搜索。而不同的镜像源的下载速度不同，过多的镜像源可能导致从较慢网速的镜像源进行下载。

因此，不宜添加过多的镜像源。

但可以设置镜像源灵活的优先权，提升搜索的效率。

conda config --set channel_priority flexible

8.conda版本更新，改善搜索能力

conda update -n base conda，conda版本更新

将conda更新到最新的版本，可以改善conda 的搜索能力。但conda更新需要操作两次更新指令。因为第一次更新较快，但并不完整。需要再输入相同的命令，第二次更新很慢，但能提供完整的conda版本更新。

# 更新conda命令
conda update -n base conda
