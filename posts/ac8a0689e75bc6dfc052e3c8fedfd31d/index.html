pdd.erp.order.syncerp打单信息同步

用途

ERP打单信息同步接口的主要用途是将ERP系统中的订单、发货、收款等业务数据同步到打单系统中，以便自动化生成快递单、发票等打印文档，并实现订单状态跟踪、快递查询等功能。该接口可以实现ERP系统与其他系统的信息互通，提高操作效率，减少重复录入数据的工作量，避免人为错误，从而提高企业运营效率和管理水平。同时，ERP打单信息同步接口还可以帮助企业对订单、库存等信息进行统一管理，实现精细化运营管理目标。

公共请求参数

参数名称 参数类型 是否必填 参数描述
type String 必填 API接口名称
client_id String 必填 POP分配给应用的client_id
access_token String 非必填 通过code获取的access_token
timestamp String 必填 UNIX时间戳，单位秒，需要与拼多多服务器时间差值在10分钟内
data_type String 非必填 响应格式，即返回数据的格式，JSON或者XML（二选一），默认JSON，注意是大写
version String 非必填 API协议版本号，默认为V1，可不填
sign String 必填 API输入参数签名结果，签名算法参考开放平台接入指南第三部分底部。

 请求示例

package com.pdd.pop.sdk.http.demo;

import com.pdd.pop.sdk.common.util.JsonUtil;
import com.pdd.pop.sdk.http.api.pop.request.PddErpOrderSyncRequest;
import com.pdd.pop.sdk.http.api.pop.response.PddErpOrderSyncResponse;
import com.pdd.pop.sdk.http.PopClient;
import com.pdd.pop.sdk.http.PopHttpClient;

public class PopClientDemo {

    public static void main(String[] args) throws Exception {

        String clientId = "your clientId";
        String clientSecret = "your clientSecret";
        String accessToken = "your accessToken";
        PopClient client = new PopHttpClient(clientId, clientSecret);

        PddErpOrderSyncRequest request = new PddErpOrderSyncRequest();
        request.setLogisticsId(0L);
        request.setOrderSn("str");
        request.setOrderState(0);
        request.setWaybillNo("str");
        PddErpOrderSyncResponse response = client.syncInvoke(request, accessToken);
        System.out.println(JsonUtil.transferToJson(response));
    }
}
