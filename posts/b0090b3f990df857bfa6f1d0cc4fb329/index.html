前言

某些协议需要调优以获得最佳性能。在某些情况下，需要调优才能绑定协议。频率调优是每个 MULTI 模块所特有的，是由于射频组件的微小变化引起的。

！Note

频率调优通常只需要在每个协议中执行一次，因为对于同一类型的多个接收机，所找到的值之间通常只有很小的差异。一旦知道，同一数值通常可以用于所有使用该协议的接收机。

需要频率调优的协议有：

  • FrSkyD (e.g. FrSky D4R and D8R, DIY RX-F801 and RX-F802 receivers)；
  • FrSkyV (e.g. FrSky V8R4, V8R7 and V8FR receivers)；
  • FrSkyX (e.g. FrSky X4R, X6R, X8R, XM+ and XSR receivers)；
  • S-FHSS (e.g. Futaba S-FHSS receivers)；
  • Corona (e.g. Corona V1 FSS, Corona V2 DSSS CR8D/CR6D/CR4D and FlyDream IS-4R/IS-4R0 receivers)；
  • Hitec (e.g. Optima, Minima, Micro and RED receivers)；
  • HoTT (e.g. Graupner receivers)；
  • 任何协议都可以使用 CC2500 射频模块来模拟 XN297。这将在协议详细信息页面上注明，例如 E010，V911S 等。

！Tip

一般的规则是，如果协议菜单中出现 RF Freq. fine tune 选项，则需要进行频率调优，并且应该进行频率调优。

始终建议进行频率调优，但特别是当：

  • 难以与接收机绑定，或连接较弱或断断续续；
  • 控制范围非常短；
  • 未接收遥测数据或(仅适用于启用遥测的接收机)。

！Tip

完成频率调优的过程，可以确保无线电和接收机将有最强连接的可能。

！Warning

如果你改变了频率调优值，最好重新绑定接收机。

1 频率调优步骤

该过程可以在串行或PPM模式下执行。串行模式更容易，因为可以实时观察效果。

！Warning

为了获得最佳效果，应该使用真正的 FrSky, Futaba, Corona, Hitec 或 HoTT 接收机进行微调。

1.1 准备工作

如果接收机没有绑定，并且协议有 RF Freq. fine tune 选项，请将 RF Freq. fine tune 值更改为 -40 或 40，然后尝试再次绑定。如果绑定仍然不成功，以正负 40 为步长继续尝试较高和较低的值，直到绑定成功。

  ！Tip

请确保你尝试 RF Freq. fine tune 值的整个范围，-127 到 +127。

关于完成绑定（Binding）过程的更多细节说明，请参见绑定文档页面。一旦接收机被绑定，你就可以继续进行微调。

1.2 串行模式微调

！Note

如果你配置了遥测功能和具有语音功能的无线电，你可以启用遥测丢失的语音警报，这样当接收机连接丢失时，你就会立即收到警报。

随着无线电被绑定到接收机上：

1) 返回到 RF Freq. fine tune 选项；

2) 降低该值，直到无线电失去与接收机的连接。记录该值(TUNE_MIN)；

3) 提高该值以恢复连接，然后继续提高该值，直到无线电再次失去与接收机的连接。记录该值(TUNE_MAX)；

4) 计算两个值之间的中值(TUNE_MIN + TUNE_MAX) / 2 = TUNE_MEDIAN；

5) 将 RF Freq. fine tune 设置为中值。

！Tip

如果你编译自己的固件版本，你可以在固件中设置微调值，从而无需在每个模型上配置它。

1.3 PPM模式微调

在 PPM 模式下进行微调，需要使用相同的协议配置多个开关位置，但使用不同的微调值，然后编译和上传固件并测试每个值。 编译-上传-测试过程需要重复几次，以获得准确的微调值。

你将需要熟悉配置和编译MULTI-Module的源代码。更多信息请参见编译固件（Compiling the Firmware）页面。

2 高级用户

编译自己的 MULTI-Module 固件的用户可以在固件中指定预先设置的频率微调值，以便它们自动应用于给定的协议。

在配置固件源时，在 _Config.h 中找到并编辑这一部分：

/*******************************/
/*** CC2500 FREQUENCY TUNING ***/
/*******************************/
//For optimal performance the CC2500 RF module used by the FrSkyD, FrSkyV, FrSkyX, SFHSS, CORONA and Hitec protocols needs to be tuned for each protocol.
//Initial tuning should be done via the radio menu with a genuine FrSky/Futaba/CORONA/Hitec receiver.  
//Once a good tuning value is found it can be set here and will override the radio's 'option' setting for all existing and new models which use that protocol.
//Valid range is -127 to +127
//Uncomment the lines below (remove the "//") and set an appropriate value (replace the "0") to enable.
//#define FORCE_FRSKYD_TUNING 0
//#define FORCE_FRSKYV_TUNING 0
//#define FORCE_FRSKYX_TUNING 0
//#define FORCE_SFHSS_TUNING  0
//#define FORCE_CORONA_TUNING 0
//#define FORCE_HITEC_TUNING  0
//#define FORCE_HOTT_TUNING   0

！Note

如果固件中设置了微调值，则无线电中的 RF Freq. tuning 值将被忽略。
