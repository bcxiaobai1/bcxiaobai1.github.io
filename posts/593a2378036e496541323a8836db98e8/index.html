程序员如何在内卷环境中脱引而出？

程序员本来是一个需要高学历，高技能的工作

但是随着“科技发展”，慢慢的自称自己为“码农”、“搬砖者”，而且一边喊着“35岁危机”，一边996的干活。身体慢慢的发胖（过劳肥），头发渐渐稀少，并且在不断内卷中。

下面通过一个故事，一起感受下，程序员从什么时候开始卷起来的。

从前

有个函数-涵涵
type add(type a, type b);
它爸希望它返回a+b。

有一天涵涵被诊断出先天过敏性疾病，给它吃666和35,它居然拉肚子了（回答了250）。

printf(add(666,35))
250

于是，它爸的兄弟也就是它的舅舅们想了一个办法

让他吃东西之前看看，这东西它会不会过敏。

type addFun(type a, type b){
	if( a!=666 && b!=35)
		return add(a,b);
	else 
		return 701;
}

大家发现这种方法简单粗暴。于是每一次佳佳被诊断出先天性疾病的时候都在前面加上一句if else。
粉丝福利

N年后

又过了N年

大家发现-涵涵的食物搭配-还可以被用来检测它亲兄妹健健的先天性疾病，于是有了

subfun(a,b){
	if(add(a,b) != addfun(a,b)){
		return sub(a,b);
	}
	else{
		return subfun(a*10,b*10)/10
	}
}

后来医生对-涵涵-和它的近亲程程以楚楚芳芳等及其他拥有相同病症的人给出的治疗方案。

有一天，有个人发现了楚楚有一个特性，每当给它的第二个食物是0的时候它都会发脾气把家给烧了。
这和它的远门祖宗球球表现的一模一样exit(0);
可是-涵涵-的上司请不起球球，因为球球太贵了。
于是上司门决定给楚楚的第二道菜是0来让楚楚烧家。
exit(0); 被替代为 div(0,0);

1万年后

科技的进步，时代的前行。医生们考古发现了涵涵的先天性疾病，想对它进行治疗（debug）。
功夫不负有心人，佳佳的病被治好了。这个时候医生们发现，佳佳的病治好了，可是依赖佳佳的健健和程程们都因为佳佳的大病初愈而而疯狂。
医生们想着，既然佳佳都能被治好，那就顺便治一下健健和程程吧。
当医生们完成手术之后，发现楚楚，芳芳，开开，萍萍好像也因为他们被治好后而疯狂。 疯狂的不仅仅是他们，还有他们的孩子们（所有使用这些函数的人）。
于是医生们日复一日，年复一年的治疗啊，发现这治下去没完没了了。

终于医生们放弃了，把治好的病人们又恢复了原状（放弃debug）。
终于世界又恢复了宁静。

10万年后

有个叫灭霸的人，它发现这样下去会有大量的资源被这些疾病缠身的人们给消耗殆尽。于是它打响了一个响指，所有和涵涵有关的人都被消灭了。灭霸和它的伙伴们开始了世界的修缮（重构）

终于在他们的执着努力下，所有被他们消灭的人的岗位都得到了新的替代。他们终于消灭了所有的已知疾病。

不知又过了多久，一个叫嘉佳的年前函数，被确诊为拥有了某先天性疾病。

结语

程序员的压力，大都来自，对预期的把控，不断缝缝补补。

历史代码千千万，更不敢轻易改动。

能够撑到使用-涵涵的食物搭配-来检测它亲兄妹健健的先天性疾病的公司才算是通过“达尔文进化论”的考核。。。
