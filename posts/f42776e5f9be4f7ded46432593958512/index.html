我来自一个普通二本，学了一个天坑的专业物联网工程，为什么说他坑呢？因为这是一个涉猎范围特别广的专业！

给大家看看我们学校开的课（时间顺序）

  1. C语言 计算机都学C语言吧正常
  2. java 这么火的java语言怎么能不学呢？
  3. 模拟电路技术 我们以后要学硬件的，模电打基础，没问题
  4. java web java都学了，不学点web怎么行呢？
  5. 数字电路技术 模电打基础，跟硬件打交道得看懂电路图吧？
  6. 单片机 这是物联网的标志吧？不学不行
  7. 嵌入式 单片机的性能有局限，嵌入式更强！
  8. java EE 企业级开发么！一点也不意外
  9. ZigBee 低功耗组网，没问题！
  10. 海量数据存储 有点不太懂了
  11. 数据挖掘 有点深度了，不太明白
  12. 人工智能 算法能听懂，但是应用很少，感觉学了个寂寞

其实我自己列举下来，发现自己学的也挺必要的，

可是还给我一种感觉就是什么都想学，什么都学不好，

学校培养我们成为了一个java工程师 ， 人工智能算法工程师 ， 嵌入式工程师 ，这个就业面似乎太广了，这导致我和我的同学都觉得自己什么都懂，但自己什么都干不好？

你和朋友讨论过的哪些有趣的技术问题？

其实我以前对计算机没有多大的兴趣，我现在为什么有兴趣了？

  • 因为计算机里有很多意想不到的有趣的题，什么背包问题啊，n皇后问题，走迷宫的问题，上台阶问题，

这些问题你解决了你会很有成就感，更有趣的是，你以为你解决了，其实你并没有解决！经过你的测试，你发现你的算法设计有缺陷，有些极端情况考虑不到位，我天，要是这种问题你也解决了，你简直要高兴一个小时！

我第一次讨论技术问题时，是我的C语言老师，他给我们出了一道运算符的题，
问题关键就是考察优先级和结合律，有的左结合有的右结合，一堆运算符，优先级还不同，

  • 我跟我一个朋友讨论了近20分钟，我记得很清楚，当时是一个互联网大会，前面大牛在那分享前沿技术，我在那看优先级，讲的啥我是一句没听，哈哈哈，别跟我学习！
