    通过可集中控制管理和监管工作的云控制台提高运营效率。通过支持云计算的自动化和修复功能简化 vCenter 生命周期管理 (sysin)，并将虚拟机置备到任何 vSphere 集群。

    在不中断工作负载的情况下，释放云优势并增强现有 vSphere 部署。激活 vSphere 混合云附加服务，以加快灾难恢复、勒索软件防护和容量规划。

  • 借助 vSphere+ Developer Services 加快创新

    将现有虚拟基础架构转变为企业级自助服务 Kubernetes 平台。使用 vSphere 多云管理实现平台运维集中化 (sysin)，以提高 Kubernetes 集群的安全性和监管能力。

常见问题解答

常见问题解答

  • vSphere+ 是 vSphere 的新版本吗？

    vSphere+ 是一种基于订阅的新产品/服务，其中包含相互交互的本地部署组件和云组件。本地部署组件包括云网关、vCenter 实例和 ESXi 主机。云组件包括面向管理员（或 IT 运维人员）和开发人员（或 DevOps）的、可增强本地部署功能的各种云服务 (sysin)，所有这些服务均可在 VMware Cloud 控制台中进行访问。云网关将您的 vCenter 实例连接到 VMware Cloud 控制台，您可以在其中集中管理本地部署基础架构并访问云服务。

  • 能否将现有 vSphere 环境和许可证升级或转换为 vSphere+？

    可以，vSphere+ 经过精心设计，可为已在使用 vSphere 的客户提供简单、无中断的升级。现有 vSphere Enterprise Plus 许可证可以轻松转换为基于订阅的授权。请联系您的 VMware 合作伙伴或销售代表，开始升级到 vSphere+。

  • vSphere+ 是 SaaS 产品/服务吗？

    vSphere+ 包括以 SaaS 形式提供的云服务。您的 vCenter 实例和 ESXi 主机仍保留在本地部署环境中。

  • 订阅 vSphere+ 的流程是什么？

    请联系您的 VMware 合作伙伴或销售代表，了解如何订阅 vSphere+。

  • vSphere+ 是否有试用版？

    请联系您的 VMware 合作伙伴或销售代表，以便在您的环境中免费试用 vSphere+。

  • 如何开始使用 vSphere+？我需要下载和安装什么？

    您可以轻松地开始体验 vSphere+。VMware 的入门电子邮件提供了设置说明，该过程大约需要一个小时。

  • 旧版 vSphere 能否由 vSphere+ 代管？

    可以，vSphere+ 支持 ESXi 6.7 或更高版本。需要将 vCenter 实例更新为最新版本。

  • 运行 vSphere+ 是否有特殊的硬件要求？

    除了基本的 vSphere+ 要求之外，vSphere+ 没有特殊的硬件要求。

  • vSphere+ 相对于 vSphere 的主要增强功能有哪些？

    利用 vSphere，您可以：

    • 访问以即服务形式 (SaaS) 提供的云服务
    • 使用云控制台实现集中管理 (sysin)
    • 创建企业级 Kubernetes 环境，无需额外付费
    • 通过订阅购买

    查看 vSphere+ 中提供的一系列详细功能特性。

  • vSphere+ 支持哪些类型的工作负载？

    vSphere+ 支持基于虚拟机的传统工作负载，以及基于容器的现代工作负载。vSphere+ 和 vSphere 支持的工作负载类型没有区别。

  • 可以购买哪些附加服务并与 vSphere+ 配合使用？

    VMware 正在开发 vSphere+ 附加服务，以加快灾难恢复、勒索软件防护、容量规划等工作。有关当前可供购买或正在开发的附加服务的列表，请联系您的 VMware 合作伙伴或销售代表。

  • vSphere+ 包括哪些开发人员服务？

    vSphere+ 包含广泛的开发人员服务，包括虚拟机服务、存储服务、网络服务、注册表服务、Tanzu Kubernetes Grid Service、Tanzu 集成服务、Tanzu Mission-Control Essentials 等。这些服务包含在 vSphere+ 中，无需额外付费。

  • vSphere+ 包括哪些管理员服务？

    vSphere+ 包括广泛的管理员服务，包括全局清单服务、事件查看服务、安全运行状况检查服务、虚拟机置备服务、生命周期管理服务以及配置管理服务等 (sysin)。这些服务包含在 vSphere+ 中，无需额外付费。查看 vSphere+ 中功能特性和说明的详细列表。

  • 在哪里可以找到 vSphere+ 主要功能特性的列表？

    查看 vSphere 产品/服务中功能特性的详细列表。

  • vSphere+ 云网关的安全性如何？它是否会使我的 IT 环境容易受到网络攻击？

    vSphere+ 通过将本地部署 vCenter 实例链接到 VMware Cloud 的安全云网关，提供云优势。vSphere+ 云网关使用行业标准传输层安全协议 (TLS) 来保护网络连接，并且不会将您的专有数据复制或共享到云端。该网关允许 VMware 启动维护和支持操作，通过应用权限分离和最低权限等传统安全概念来确保这些操作的安全。

  • 升级现有 vSphere 环境时，vSphere+ 是否要求我将工作负载、vCenter 实例或任何其他 vSphere 基础架构迁移到云端？

    您的任何工作负载或本地部署 vSphere 基础架构都不会迁移到云端。您的工作负载将继续在 ESXi 主机上本地运行。借助 vSphere+，您可以从云控制台集中管理本地部署环境，并通过云服务增强管理。

    如果您选择将本地部署工作负载迁移到云，vSphere+ 将使迁移过程变得更容易。您可以使用 vSphere+ 云网关中内置的混合链接模式功能将工作负载迁移到 VMware Cloud on AWS。

  • 如果与云的连接断开，会发生什么情况？vSphere+ 会停止运行吗？

    如果您失去与云的连接，只有对云服务和云控制台的访问会受到影响。vCenter 实例、ESXi 主机和工作负载将继续正常运行，并且可从 vCenter（通过 vSphere Client）进行管理。

  • 我需要学习哪些新工具？

    要获得 vSphere+ 的优势，唯一需要的额外工具是可通过 Web 浏览器访问的 VMware Cloud 控制台。

  • 如何访问 VMware Cloud 控制台？

    可通过以下网址访问 VMware Cloud 控制台：https://vmc.vmware.com。登录后，用户可以查看和订阅可用的 VMware 服务，包括 vSphere+。
