MEMO通证经济学

  1. MEMO 协议简介

MEMO 是 MEMO Labs 开发的新一代区块链去中心化云存储协议。我们的使命是为 Web3.0 构建可靠的基础架构。为了实现高可扩展性和可用性，MEMO 对数据分层、验证、容错和修复机制进行了大量创新。MEMO以全新架构和多项创新在区块链云存储上取得技术突破。

MEMO设计了三种用户角色：User、存储空间用户；Provider，存储空间提供者；和 Keeper，协调经理。在智能合约的驱动下，三个相互关联的角色相互制约。

MEMO 可以分为三个基于角色的功能层：结算、验证和存储。结算层通过聚合订单信息并将每个订单的金额发送到存储节点Provider来处理链上结算。验证层是在链下进行的。Keeper 节点向 Provider 节点发起挑战，验证证明结果，并决定向 Provider 节点颁发提现凭证。验证层上的所有进程都会经过同层节点，通过拜占庭容错共识。存储层由海量分散的 Provider 节点组成，存储真实的 User 数据，并定期向验证层提交存储证明。

  1. TOKEN 设计

2.1定义

MEMO 是一种 MEMO 协议令牌，用于驱动操作和扩展整个 MEMO 存储协议。在该协议中，为了赚取收入，Keeper 和 Provider 需要向 MEMO 进行质押，而 User 使用 MEMO 代币来支付数据存储和检索费用。MEMO 将根据订单量增加代币的发行，以激励 Keeper、Provider 和长期代币持有者。MEMO 只是一种使用中的代币，也是一种权益。这使 MEMO 代币持有者能够通过参与我们的治理从不断发展的 MEMO 生态系统中受益。

2.2 原则

从技术上讲，MEMO Token 旨在打造一个大规模的数据存储池，通过我们的公开验证机制、加密算法、多级容错和修复机制保证数据的安全、可靠和隐私，同时有效降低链上负载和通过我们的分层存储扩展性能。

在我们的 TOKEN 设计中，MEMO 旨在为我们的用户和生态系统参与者创造价值，提高生产力，并激励生态增长。我们经济模型的原则是与成本成比例的最公平的参与者激励。经济模型的逻辑应该由代码支配，以消除人为控制。经济模型应该激励有利于协议的参与者行为，并满足免费激励。

三、流通机制

3.1 MEMO 经济流通

TOKEN 将在 User、Keeper、存储节点 Provider 以及开发者、投资者和代币持有者之间流通。首先，User 需要购买 MEMO 代币来支付存储服务费用，Keeper 和 Provider 需要购买 MEMO 代币作为质押物，然后才能提供服务并获得收益。用户为存储服务支付的 MEMO 代币将发送给 Keeper 和 Provider。同时，通过增加代币发行作为对池中 MEMO 的质押，将激励 Keeper、Provider 和长期持有者促进生态发展。所以 MEMO 的生产和流通是一个消费和再生产的循环，随着存储逐渐规模化和代币价值升值。

3.2 MEMO使用场景

MEMO 协议中的 MEMO（MetaMEMO）有两种使用场景：作为质押物和作为存储服务的支付。质押通常与 Keeper 和 Provider 相关，而付款则与用户和开发人员有关。

质押：为了更好的用户体验，MEMO 协议规定了 MEMO TOKEN 的质押作为 Keeper 和 Provider 的资格标准。目的是限制和规范 Keeper 和 Provider 的行为，倡导诚实可靠的服务。

支付：用户，即最终消费者或开发者，使用 TOKEN 支付数据存储、检索和下载服务。TOKEN 支付最终将按照给定的顺序分配给 Provider 和 Keeper，作为提供数据存储和管理服务的收入。因此用户或开发者需要在使用存储服务前购买 MEMO TOKEN 并在签署订单前将其转移到订单池中才能使用 MEMO 存储服务。

MEMO 也代表了整个协议的权益。因此，MEMO 持有者可以将 MEMO 代币质押到质押池中，以获得额外的代币作为收入。同时，MEMO 代币持有者将能够通过未来的治理机制参与协议的治理。

  1. 代币分配

4.1 代币经济学

MEMO 的初始代币供应上限为 600,000,000，当前没有通货膨胀。预计该协议将如何谨慎地在一段时间后纳入适度的通货膨胀以激励基于社区的充满活力的参与者。

这些不同的代币分配被锁定，以确保我们的投资者相信我们有最好的意图并保持对项目的长期愿景。这些代币通过智能合约进行保护，并将支付给 MEMO 团队、投资者、合作伙伴和顾问、社区，以便他们决定将其用于诸如向开发人员和贡献者提供赠款、调整协议分发和生态等目的用于 MEMO 生态系统的研发基金，具有流动性和 Staking 分配，用于交易所流动性和 Staking 奖励。这有助于保持他们的利益一致。

4.2 初始代币发行机制

代币分配分为初始分配和附加激励分配。首次分配的股份和释放期限如下：

投资者（15%）：15% 的代币分配给投资者，其中 5% 用于种子轮投资者，他们帮助建立了协议，他们的利益在 6 个月的悬崖和 18 个月的发布时间表的长期范围内保持一致以及上市前10%的代币释放计划；9% 的私人投资者在上市前有 15% 的代币释放计划，6 个月的悬崖和 18 个月的线性释放计划；剩下的 1% 给公众投资者，100% 解锁。

团队（14%）：还有 14% 的分配给团队，通过智能合约授予 5 年发布时间表。

合作伙伴和顾问 (7%)：合作伙伴和顾问也分配了 7%，并通过智能合约授予了为期 5 年的发布时间表。

生态基金（15%）：生态基金占 15%，解锁周期为 5 年，通过智能合约授予。

社区金库（49%）：49% 分配给社区金库，社区可以决定将其用于向开发者和贡献者提供赠款、调整协议分发机制。

五、生态发展

5.1 基础

MEMO 基金会主要用于生态系统的开发、标记和社区维护。同时，部分资金投入用于促进生态发展，维持基金会的长期可持续运营。

MEMO基金会承担以下职责：

● 组织开发团队或外包任务以实施 MEMO 协议及其更新。

● 支持和资助MEMO 生态应用。

● 持续为MEMO 社区的长期运营做出贡献。

MEMO 基金会有权发起有关系统治理的提案。社区对其最终实施有决定权。

基金会可发起以下提案，包括但不限于：

● 修改系统的经济参数。

●提出技术更新。

● 惩罚守门员的不当行为或不作为。

● 惩罚提供者的不当行为或不作为。

5.2 社区生态

MEMO协议作为区块链去中心化存储解决方案的发展，离不开我们社区的支持。MEMO 基金会将积极组织和建设生态管理者、开发者、代币持有者等各类社区团体，以鼓励我们的协议和生态在多方面健康稳定发展。

5.3 守护者

在加入我们的协议之前，Keeper 必须保证足够的存款。

保管人必须履行以下义务：

● 质押不得少于规定的保证金数额。

● 保证长期在线活动，维护验证层历史数据。

● 验证Provider的存储证明并向符合拜占庭容错共识的Provider发出提现指令。

● 提供商应为其数据丢失安排数据修复过程。

同时，Keeper 享有以下权益：

● 一定比例的管理订单收益。

● 协议对质押存款的额外激励。

处罚可能适用于任何不当行为或不作为，例如从押金中扣除，并且不再从服务提供中获得收入。

5.4 提供者

在加入我们的协议并从与客户签署的订单中获得收益之前，提供商必须保证足够的存款。

提供者必须履行以下义务：

● 质押不得少于规定的保证金数额。

● 确保基于订单的标准数据存储、数据可靠性和可用性。

● 及时向验证层提交存储证明。

同时，提供者有权享受以下福利：

● 从订单计费中逐步付款。

● 协议对质押存款的额外激励。

任何未能履行义务（例如数据丢失或逾期提交证明）都可能导致处罚，包括从押金和订单转移中扣除。

5.5 开发者

开发者可以在他们的项目中采用 MEMO 协议。更多使用MEMO存储服务的项目将带来更多的用户和存储需求。

基金会将通过开发者活动筛选对 MEMO 协议有价值的贡献者，并通过代币激励开发者孵化项目。开发者必须履行以下义务：

● 维护他们的项目。

● 与最新的备忘录协议保持同步。
