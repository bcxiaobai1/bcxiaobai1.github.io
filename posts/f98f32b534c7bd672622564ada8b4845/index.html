OpenAI深夜放大招，GPT4 API全面开放并弃用一系列旧模型

GPT-4 API 现已向所有付费 OpenAI API 客户开放。GPT-3.5 Turbo、DALL·E 和 Whisper API 现已普遍可用，我们宣布了一些旧型号的弃用计划，这些型号将于 2024 年初退役。 

✅ GPT4 API面向付费用户开放，不需要再额外申请,并且具有8K上下文，计划本月底完成

✅ 从之前的提供自由格式的文本提示，转向更结构化的提示界面，可以提供更好的结果。基于聊天的范例已被证明是强大的，可以处理绝大多数以前的用例和新的对话需求，同时提供更高的灵活性和特异性。

✅ 弃用旧模型，需要迁移到新模型，2024 年 1 月 4 日开始，旧的型号将不再可用，并将替换为新的模型。

✅ 弃用旧的嵌入模型，旧嵌入模型的用户需要在 2024 年 1 月 4 日之前迁移到 text-embedding-ada-002

✅ 弃用 Edit API, Edits API 及其相关模型（例如 text-davinci-edit-001 或 code-davinci-edit-001）的用户需要在 2024 年 1 月 4 日之前迁移到 GPT-3.5 Turbo。

从今天开始，所有付费 API 客户都可以访问 GPT-4。三月份，推出了 ChatGPT API，本月早些时候，我们发布了基于聊天的模型的首次更新。我们设想未来基于聊天的模型可以支持任何用例。今天，我们宣布了针对旧版 Completions API 的弃用计划，并建议用户采用 Chat Completions API。

GPT-4 API 全面可用

GPT-4 是我们最有能力的模型。自 3 月份以来，数以百万计的开发人员请求访问 GPT-4 API，并且利用 GPT-4 的创新产品的范围每天都在增长。如今，所有具有成功支付历史的现有 API 开发人员都可以访问具有 8K 上下文的 GPT-4 API。我们计划在本月底之前向新开发人员开放访问权限，然后根据计算可用性开始提高速率限制。

基于这些模型的稳定性和适合生产规模使用的准备，我们还普遍提供 GPT-3.5 Turbo、DALL·E 和 Whisper API。我们正在努力安全地启用 GPT-4 和 GPT-3.5 Turbo 的微调，并预计该功能将在今年晚些时候推出。

从文本转向聊天对话模式

我们在 3 月份推出了 Chat Completions API，它现在占我们 API GPT 使用量的 97%。

最初的 Completions API 于 2020 年 6 月推出，旨在提供自由格式的文本提示，以便与我们的语言模型进行交互。从那时起，我们了解到，通过更结构化的提示界面，我们通常可以提供更好的结果。基于聊天的范例已被证明是强大的，可以处理绝大多数以前的用例和新的对话需求，同时提供更高的灵活性和特异性。特别是，聊天完成 API 的结构化接口（例如系统消息、函数调用）和多轮对话功能使开发人员能够构建对话体验和广泛的完成任务。它还有助于降低即时注入攻击的风险，因为用户提供的内容可以在结构上与指令分离。

我们计划继续在这个方向上投入大部分平台工作，因为我们相信它将为开发者提供越来越强大且易于使用的体验。我们正在努力快速缩小聊天完成 API 的最后几个剩余差距，例如完成令牌的日志概率和提高可操纵性以减少响应的“闲聊”。

Completions API 中弃用旧模型

作为我们对 Chat Completions API 增加投资以及优化计算能力的努力的一部分，我们将在 6 个月内淘汰一些使用 Completions API 的旧模型。虽然此 API 仍可访问，但从今天开始，我们将在开发人员文档中将其标记为“旧版”。我们计划未来的模型和产品改进将重点放在 Chat Completions API 上，并且没有计划使用完成 API 公开发布新模型。

从 2024 年 1 月 4 日开始，旧的型号将不再可用，并将替换为以下型号：

使用基本 GPT-3 模型（ada、babbage、curie、davinci）的稳定模型名称的应用程序将于 2024 年 1 月 4 日自动升级到上面列出的新模型。新模型也将在未来几周内提前提供通过在 API 调用中指定以下模型名称进行测试：ada-002、babbage-002、curie-002、davinci-002。

使用其他较旧模型（例如 text-davinci-003）的开发人员需要在 2024 年 1 月 4 日之前通过在 API 请求的“model”参数中指定 gpt-3.5-turbo-instruct 来手动升级其集成。gpt-3.5-turbo-instruct 是一种 InstructGPT 风格的模型，其训练方式与 text-davinci-003 类似。这个新模型是 Completions API 中的直接替代品，并将在未来几周内提供早期测试。

希望在 2024 年 1 月 4 日之后继续使用其微调模型的开发人员将需要在新的基本 GPT-3 模型（ada-002、babbage-002、curie-002、davinci-002）或更新版本之上微调替换模型型号（gpt-3.5-turbo、gpt-4）。一旦今年晚些时候此功能推出，我们将优先为之前对旧型号进行微调的用户提供 GPT-3.5 Turbo 和 GPT-4 微调。我们承认，迁移根据您自己的数据进行微调的模型具有挑战性。我们将为之前微调模型的用户提供支持，以使这一过渡尽可能顺利。

在接下来的几周内，我们将联系最近使用过这些旧模型的开发人员，并在新的完成模型准备好进行早期测试后提供更多信息。

弃用旧的嵌入模型

旧嵌入模型（例如 text-search-davinci-doc-001）的用户需要在 2024 年 1 月 4 日之前迁移到 text-embedding-ada-002。我们于 2022 年 12 月发布了 text-embedding-ada-002，并且我们发现它比以前的型号更强大且更具成本效益。目前，text-embedding-ada-002 占所有嵌入 API 使用量的 99.9%。

我们认识到这对于使用旧模型的开发人员来说是一个重大变化。结束这些模型并不是我们轻易做出的决定。我们将承担用户使用这些新模型重新嵌入内容的财务成本。我们将在未来几天与受影响的用户联系。

弃用编辑 API

Edits API 及其相关模型（例如 text-davinci-edit-001 或 code-davinci-edit-001）的用户需要在 2024 年 1 月 4 日之前迁移到 GPT-3.5 Turbo。Edits API Beta 版是早期版本探索性 API，旨在使开发人员能够根据说明返回提示的编辑版本。在开发 gpt-3.5-turbo 和 Chat Completions API 时，我们考虑了编辑 API 的反馈，现在可以将其用于相同的目的
