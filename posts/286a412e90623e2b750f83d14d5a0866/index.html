1.1.1 课题研究背景
2015年10月，中共十八届五中全会中强调，“十三五”发展目标要尽全力实现，我们要树立创新意识，以绿色环保、开发共享的理念不断进步，“坚持共享经济，着力增进人民福祉”[1]。我们每个人都要参与其中，尽我们最大的力量去实现要求，绿色环保的成果是大家所共享的。“共享”一词随着互联网的兴起，呈现出不可阻挡的发展态势，共享理念所带来的不仅是理念的改变，更是为人们提供了一种完全不同于过往的全新生活方式。
中国2018年共享经济发展报告提到，中国为全球共享领域的贡献是不容忽视的。创新发展是共享经济的基础，共享经济更是动力源泉，全球生产、生活、思考方式正在因此而改变，共享经济在中国的发展周期较短，但在市场、创新、国际影响力等方面都拥有不容忽视的能力[2]。
我们在平时的互联网使用中也可以清楚的看到，之前所使用的闲置交易网站已经不能满足网民对于闲置交易的需求，各大闲置交易平台如春笋般增长，其中，闲鱼闲置交易是阿里巴巴结合淘宝网推出的一款闲置交易平台，目前市场占有量较大，但是我们不能忽视的是，更多具有针对性的闲置交易平台开始出现，如瓜子二手车、人人二手车交易网、孔夫子二手书等网站正在不断完善，想在特定的领域下获得成功，闲鱼交易平台针对的面太广而且有一些比较细的方面涉及不到，给了其他的互联网产品以成功的契机。
本设计以搭建一个方便校园内处理个人闲置物品的交易平台为目的，针对大学生这一特定人群以及适合他们的互联网产品，首先在于受众面极大，众所周知每年我国的毕业生都持续增长，据预测2018年的毕业生更是高达九百多万，校园闲置物品处理主要针对的正是高校毕业生。多数大学生处理闲置的方式都是直接扔掉，造成资源的极大浪费，如果出现一款针对学校的互联网产品，那么会适当的增加二手闲置商品的利用率，这也符合共享经济的特点。
1.1.2 课题发展及现状
当代大学生对新鲜事物始终保持着活力，对于各类先进产品的追逐尤其明显，由于大部分大学生还无法做到经济独立，通过卖掉闲置来获取经济以便于购买新的物品成为了一个可行的选择。支付宝权威统计显示，2016年中国高校学生在互联网（主要是支付宝）的消费金额人均以及达到了40839[3]。虽然数据是两年前，但是由于人们的消费水平在两年内不断增长，所以大学生的消费水平只会持续的提高，其实大学生使用支付宝很大一部分的消费在于淘宝购物，特别是开学以及商家活动期间，学校里的快递点已经到了挤不进去的地步，但是每一次的购买新物品那么就会有很多的闲置物品的搁置，有些同学觉得丢了可惜但是又没有恰当的处理方式，校园内也存在一些家境不是那么富裕的情况在考虑购买闲置物品，所以就造成了闲置利用率很低的现状。据实际了解，目前高校内处理闲置物品的互联网方式主要是在贴吧、微博，有高校会为毕业生离校举办跳蚤市场，但是还是存在浏览不方便的问题，很多同学渴望有一款简洁方便浏览的系统平台进行闲置物品的出售或购买，这也成为了本系统的发展动力。
互联网产品针对校园生活也在不断的推陈出新，目前已知关于校园的平台的有“校园跳蚤街”、“淘二淘”、“校园淘书”（图1-1）等。

但是目前针对校园的互联网产品都存在同样的弊端，那就是宣传力度不够，宣传广告过多，针对的面太少不能满足大学生对于各类闲置购买的需求，不够吸引大学生使用，但也存在一些校园闲置交易平台界面交互过于繁琐，导致用户使用系统时不知道从何下手。论文设想的是以几个学校为受众面，以良好的界面交互。实用的功能来吸引少量的用户，以此为基点发散出去获取更多的使用量，是我们可以尝试的一个方向。

1.1.3 研究意义与目的
本系统设计的目的旨在让高校中的同学们可以有效的处理自己闲置的物品，以达到共享、环保、绿色的目的，也让需要这些闲置的同学可以找到自己想要的东西，达到物尽其用而不是浪费，具体意义有如下几点：
(1) 构建互联网闲置资源再利用平台，解决校园闲置因为场地、时间等因素造成的利用率低等问题，丰富校园文化，提高同学们的资源利用意识。
(2) 引导学生合理的消费意识，一定程度的改变目前学生不大合理的生活方式，提高闲置物品的使用、利用价值。
(3) 让闲置物品真正做到物尽其用，闲置售卖的同时也帮助校园内勤工俭学的同学可以有更好的选择，特别是一些几乎完好无缺的闲置。
1.2 相关技术简介
1.2.1 Thinkphp
ThinkPHP是一款在PHP语法的基础上二次开发的框架, 框架使用的是Apache源协议,框架开发的目的是为了简便化应用开发。TP是在MVC基础上开发的应用框架, 主要的工作是让数据进行通信，在前端编码以及对PC应用的整合，使开发速度有一个明显的提升，代码的重复率也得到了减少。在使用TP框架时，RBAC、URL、AJAX、CURD等各种优势特性会让我们的开发变得简单 [4]。
1.2.2 Wampserver
在以往的PHP开发过程中，环境配置需要十分繁琐复杂的过程，我们需要安装多个组件，Wampserver的优势在于其是PHP开发过程需要用到工具的集成环境，可以免去复杂的工具安装步骤，让我们有更多的时间投入到开发中。Wamp是Windows+Apache+Mysql+Php的缩写，我们只需要进行很基础的学习就可以掌握软件的使用，软件完全免费，让开发者可以使用界面化的形式对自己的项目进行管理，在网页数据Localhost即可直接访问本地项目 [5]。

1.2.3 Bootstrap
Bootstrap框架是一款基于CSS、JS的前框框架，相比于传统的前框框架，BS提供了一套自己的JS/CSS类库供开发者调用，不需要再自己编写繁琐的Javascript代码，使得我们的开发变得简单快捷。BS框架相对于其他传统前端开发还具有以下特性：
1.CSS代码完整，已经打包成各种类供开发者使用；
2.响应式栅格方式让我们无论在PC或是手机都能获得良好的界面交互；
3.可以直接采用BS框架结合PhoneGap进行移动应用的开发[6]。
1.2.4 Navicat数据库管理工具
Navicat是一款针对Mysql数据库管理的可视化图形软件，以更安全且便捷的方式展示给用户以数据库数据的操作，用户在一些特定的情况下也可以直接使用Navicat进行数据库的命令行编程，软件功能强大且基本功能的使用免费，可以满足大多数技术人员对于数据库的管理要求，也可满足中小型企业对于数据库的管理，数据传输稳定，可以直接连接Windows或Linux服务器进行后台数据的管理[7]。

本章对系统的背景、现状、研究意义以及系统用到的相关技术进行相关概述，了解完这些之后，可以对系统的背景有更好的理解。
2 系统可行性分析
2.1 系统经济可行性
经济可行性指的是投入与产出两者的关系，一款软件前期的投资包括技术人员以及开发设备的购买，中期主要有互联网推广以达到用户量的累积，后期的投资主要有系统维护、升级的花销。系统经济可行性的研究对后期系统的开发具有决定性意义, 投资是否合理，资金规划是否如期进行都直接影响后期的工作进行，主要表现在以下两个方面[8]。
（1）投资回报：本系统主要针对的是大学生，需要在大学生群体中形成一定的影响力，需要具有一定的用户基础以及使用量。在运营后期，可以结合学校周边商户进行活动的开展，如发布关于商家信息的广告等活动。随着用户的不断增加，在学生之间形成良好的口碑之后，会吸引到更多的商家与我们合作，后期的利润将会很可观[9]。
（2）社会效益：本系统是一个针对大学生闲置物品处理的平台，以达到资源最大化利用为目的，培养大学生更理智和节约的消费观念。相对于传统的跳蚤市场，使用互联网产品打破了地点和时间的局限，也对贴吧、微博信息杂乱的现状有了一个良好的解决方案。
2.2 环境可行性
（1）网络环境：现如今，高校互联网的普及率十分巨大，大部分学校都具备独立的校园网络，校园网将学校行政与教学连接起来以便于特定信息的共享，提高工作的效率。随着高校不断的扩招以及高校信息化工程的展开，高校网络建设的步伐将会越来越快。
（2）二手市场环境：大学生在消费观念上具有一定的共识，对处理二手商品的态度上显得不是很果断，特别是毕业季，毕业季是闲置物品处理最大的关卡，就拿广西师范大学来说，学校里面二手市场只在毕业季的时候体现在微博和贴吧上，具规模的二手市场很少见，同学们的心态是能卖则卖，卖不出去就丢掉，造成了资源的极大浪费。如果能出现一款专门针对二手交易系统平台，就会极大的方便同学们对闲置进行处理。
2.3 技术可行性
利用Wampserver框架搭建网站项目具有一定的可行性，加上Thinkphp技术在国内已经比较成熟，在设计客户端方技术方面已经逐渐清晰，采用现有的Bootstrap框架进行客户端的搭建可以最快速的进行客户端前端的构造。
后台管理系统采用Thinkphp技术，TP技术优势在于可以直接利用PHP语句插入到HTML，类似于JSP、ASP语言，代码易于理解，技术员只需了解基础的HTML、CSS、JS及PHP语法知识即可根据相应的操作进行代码的编写。
2.4 系统用户定位及行为分析
2.4.1 系统用户定位
使用本系统的用户主要是高校学生，对学校老师也具备使用价值。如今大学生对于智能手机的使用已经十分熟练，加之电子商务在高校内的流行，所以使用本产品几乎没有难度，但系统的功能也不能过于复杂，应该要做到浅显易懂，可以考虑突出学校的特色。
2.4.2 系统行为分析
本节内容是分析用户使用本系统时的行为，了解用户使用产品的初衷，以达到优化系统内容的目的，本系统的用户有以下几类：
（1）想出售闲置的卖家，此类用户手中的闲置具有时效性，如考研复习资料等一些每年都会更新但也有利用价值的的物品，或者是一些对自己用处不大，但对其他用户具有吸引力的物品，想要通过闲置物品交易平台对物品进行处理。 （2）需要购买闲置的买家，买家有购买需求，但由于购买新物品的花销过大，价值不高，所以购买闲置物品的需求就会产生。
（3）没有具体目的的买家，此类用户对于购买的东西没有局限性，但是喜欢在电子商务系统上寻找适合自己的物品，遇到合适的就会产生购买，针对性没有上面两种大。
目前针对系统用户行为的分析就是以上三种，以此为基础我们来设计一些功能来留住用户，如针对没有具体的买家，我们可以根据买家平时浏览搜索的习惯来设计算法以方便他们使用。
2.5 本章小结
本章进行了系统经济、技术、环境的可行性分析，明确了哪些用户会使用本系统，从而对系统应该具备的功能有一个大致的轮廓，可行性分析工作的质量是我们需要保证的，因为可行性分析不仅仅在系统设计前期产生作用，在设计后期有着不可估计的意义。

$(function () {
    //加载弹出层
    layui.use(['form','element'],
    function() {
        layer = layui.layer;
        element = layui.element;
    });

    //触发事件
  var tab = {
        tabAdd: function(title,url,id){
          //新增一个Tab项
          element.tabAdd('xbs_tab', {
            title: title 
            ,content: '<iframe tab-id="'+id+'" frameborder="0" src="'+url+'" scrolling="yes" class="x-iframe"></iframe>'
            ,id: id
          })
        }
        ,tabDelete: function(othis){
          //删除指定Tab项
          element.tabDelete('xbs_tab', '44'); //删除：“商品管理”
          
          
          othis.addClass('layui-btn-disabled');
        }
        ,tabChange: function(id){
          //切换到指定Tab项
          element.tabChange('xbs_tab', id); //切换到：用户管理
        }
      };


    tableCheck = {
        init:function  () {
            $(".layui-form-checkbox").click(function(event) {
                if($(this).hasClass('layui-form-checked')){
                    $(this).removeClass('layui-form-checked');
                    if($(this).hasClass('header')){
                        $(".layui-form-checkbox").removeClass('layui-form-checked');
                    }
                }else{
                    $(this).addClass('layui-form-checked');
                    if($(this).hasClass('header')){
                        $(".layui-form-checkbox").addClass('layui-form-checked');
                    }
                }
                
            });
        },
        getData:function  () {
            var obj = $(".layui-form-checked").not('.header');
            var arr=[];
            obj.each(function(index, el) {
                arr.push(obj.eq(index).attr('data-id'));
            });
            return arr;
        }
    }

    //开启表格多选
    tableCheck.init();
      

    $('.container .left_open i').click(function(event) {
        if($('.left-nav').css('left')=='0px'){
            $('.left-nav').animate({left: '-221px'}, 100);
            $('.page-content').animate({left: '0px'}, 100);
            $('.page-content-bg').hide();
        }else{
            $('.left-nav').animate({left: '0px'}, 100);
            $('.page-content').animate({left: '221px'}, 100);
            if($(window).width()<768){
                $('.page-content-bg').show();
            }
        }

    });

    $('.page-content-bg').click(function(event) {
        $('.left-nav').animate({left: '-221px'}, 100);
        $('.page-content').animate({left: '0px'}, 100);
        $(this).hide();
    });

    $('.layui-tab-close').click(function(event) {
        $('.layui-tab-title li').eq(0).find('i').remove();
    });

    //左侧菜单效果
    // $('#content').bind("click",function(event){
    $('.left-nav #nav li').click(function (event) {

        if($(this).children('.sub-menu').length){
            if($(this).hasClass('open')){
                $(this).removeClass('open');
                $(this).find('.nav_right').html('');
                $(this).children('.sub-menu').stop().slideUp();
                $(this).siblings().children('.sub-menu').slideUp();
            }else{
                $(this).addClass('open');
                $(this).children('a').find('.nav_right').html('');
                $(this).children('.sub-menu').stop().slideDown();
                $(this).siblings().children('.sub-menu').stop().slideUp();
                $(this).siblings().find('.nav_right').html('');
                $(this).siblings().removeClass('open');
            }
        }else{

            var url = $(this).children('a').attr('_href');
            var title = $(this).find('cite').html();
            var index  = $('.left-nav #nav li').index($(this));

            for (var i = 0; i <$('.x-iframe').length; i++) {
                if($('.x-iframe').eq(i).attr('tab-id')==index+1){
                    tab.tabChange(index+1);
                    event.stopPropagation();
                    return;
                }
            };
            
            tab.tabAdd(title,url,index+1);
            tab.tabChange(index+1);
        }
        
        event.stopPropagation();
         
    })
    
})
