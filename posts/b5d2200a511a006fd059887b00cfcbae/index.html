VMware vSphere 8 发布（含下载）

昨日（2022.08.30）VMware 宣布正式发布 VMware vSphere 8。

介绍 VMware vSphere 8

企业工作负载平台 VMware vSphere 8 将云的优势带入本地工作负载，通过 DPU 和 GPU 提升性能，并通过企业就绪的集成 Kubernetes 运行时加速创新。让我们更详细地了解一下。

vSphere 8 通过 VMware vSphere Distributed Services Engine 向企业引入数据处理单元，开创了异构计算的新时代。vSphere Distributed Services Engine 是现代应用程序云基础架构发展的下一步，其中运行基础架构服务的管理权分布在 CPU 和 DPU 之间。

vSphere Distributed Services Engine 将云基础架构现代化为由 DPU 支持的分布式架构，以：

  • 通过加速网络功能满足现代分布式工作负载的吞吐量和延迟需求
  • 通过为工作负载提供更多 CPU 资源，提供最佳的基础架构性价比
  • 通过集成的 vSphere 工作流降低 DPU 生命周期管理的运营开销

vSphere 分布式服务引擎保留了客户熟悉的现有第 0 天、第 1 天和第 2 天 vSphere 体验。领先的芯片供应商（NVIDIA 和 AMD）的多种 DPU 和 OEM（戴尔、HPE）的服务器设计都支持它。

分布式服务引擎卸载并加速 DPU 上的 vSphere Distributed Switch 和 NSX 网络，未来还会提供其他服务。因此，这将立即使运行需要高网络带宽和快速缓存访问（例如内存数据库）的应用程序的客户受益。

我们在支持 DPU 的主机上运行 Redis 的内部基准测试研究实现了 36% 的吞吐量提高，同时事务延迟减少了 27%。在另一种情况下，启用 DPU 的主机实现了与非 DPU 系统相似的性能，但 CPU 内核减少了 20%。这些强大的结果展示了 vSphere 8 如何帮助客户降低总计算成本并提高工作负载性能。

此外，借助 vSphere 8，IT 管理员可以利用集成的 vSphere 工作流来管理生命周期和监控性能，从而减少基础架构中运行 DPU 的运营开销。

我们很高兴地注意到，我们的技术合作伙伴已经产生了市场就绪的端到端解决方案。今天，戴尔科技 的 SmartDPU 软件解决方案基于 VxRail TM 可选择 AMD Pensando 和 NVIDIA 的 DPU。HPE 带有 VMware vSphere® Distributed Services Engine™ 的 HPE ProLiant 基于 AMD Pensando 的 DPU 的 我们也对联想和英特尔等其他合作伙伴在为市场带来更多解决方案方面取得的快速进展感到兴奋，为我们的客户提供更多选择。

在提高工作负载性能方面，vSphere 8 并不止于此。vSphere 8 通过在直通模式下增加对多达 32 个 NVIDIA GPU 设备的支持，改进了 AI/ML 模型的训练时间，并为最苛刻和最复杂的模型实现了无与伦比的性能水平——与 vSphere 7 相比增加了 4 倍。

此外，AI/ML 开发团队现在可以实现更高级别的可用 GPU 资源可扩展性，每个 VM 支持多达 8 个 vGPU——与 vSphere 7 相比增加了 2 倍。

虽然 Kubernetes 作为事实上的容器编排技术已被广泛采用，但 IT 组织需要一种简单易用的方法来管理容器和 VM。这就是 VMware 创建了原生内置于 vSphere 中的简化 Kubernetes 管理体验的原因。借助 vSphere 8，VMware 提供 了 VMware Tanzu Kubernetes Grid 2.0，旨在帮助 IT 团队和开发人员解决敏捷开发环境日益复杂的问题。Tanzu Kubernetes Grid 的最新版本为集群创建、开源 API 对齐和改进的应用程序生命周期管理功能增加了新的灵活性和控制。

DevOps 团队花费大量时间设置 Kubernetes 集群。即使在基础设施服务易于访问的情况下，它们也是为满足 IT 管理员的需求而设计的，不一定与开发环境集成。因此，开发人员要么依靠 IT 管理员来提供开发人员服务，要么依靠独立的基础设施孤岛来满足他们的需求。借助 vSphere 8，DevOps 团队现在可以从新的云消费接口服务轻松访问 IaaS 服务（例如配置虚拟机、网络、设置 Tanzu Kubernetes Grid 集群）。云消费界面通过直观的 UI 和开发人员友好的 API 简化了 vSphere 资产中的基础架构设置，从而腾出时间用于实际开发工作。

通过云消费接口配置基础设施

借助 vSphere 8，可以跨 3 个集群部署 TKG 可用区，从而提高容器化工作负载对基础架构故障的整体弹性。由于名为 Cluster Class 的新 Cluster API 功能，简化了集群创建成为可能。用户现在可以从一个地方创建、扩展、升级和删除集群。配置、升级和操作多个 Kubernetes 集群现在是一个简化的声明式过程。引入了基于 Carvel 的工具，使应用程序开发人员和平台运营商可以自信地在 Kubernetes 上进行构建。

今年 6 月， 我们推出了 vSphere 系列的新产品 VMware vSphere+，它结合了行业领先的云基础架构技术、企业级 Kubernetes 环境和高价值云服务，将现有的本地部署转变为 SaaS启用的基础设施。vSphere 8 将这些优势进一步提升了几个步骤。

借助 vSphere 8，IT 管理员可以部署以下附加云服务，以保护工作负载并优化基础架构，而且还会有更多。

VMware Cloud Disaster Recovery 附加服务通过保护任务关键型工作负载免受灾难和勒索软件的影响，提高了它们的弹性。

使用 VMware Cloud Disaster Recovery 保护工作负载

VMware vRealize Operations 附加服务为您的基础架构提供容量规划和优化，其大小适合您当前和未来的工作负载需求。

轻松获取有关 CPU、内存和存储消耗的详细信息

IT 管理员花费大量时间升级和维护基础架构。尽管定期维护操作有助于确保正常运行时间和可用性，但它们会占用运行关键业务应用程序的时间。vSphere 8 通过预先准备 ESXi 映像下载并在主机上执行同步升级来优化维护窗口，使团队能够更快地恢复到常规操作。

随着本地和边缘工作负载的增长，初始放置和迁移是帮助基础架构团队最大限度地提高服务可用性、平衡利用率和最大限度地减少停机时间的关键方面。vSphere 8 对 vSphere Distributed Resource Scheduler 和 vMotion 进行了重大升级。分布式资源调度程序现在将工作负载内存使用情况纳入布局决策。它现在可以通过考虑工作负载的内存需求来更优化地放置工作负载。

vMotion 现在支持迁移在支持英特尔可扩展 I/O 虚拟化 (英特尔 SIOV) 的主机上运行的虚拟机。工作负载现在可以同时享受 SIOV 直通性能和跨 vSphere 基础架构的移动性的优势。

根据 IDC 的一项研究，到 2022 年，全球 65% 的 GDP 将实现数字化 (4) 。IDC 研究指出，全球数据领域预计将在 2022 年至 2026 年间翻一番 (5) 。随着计算的足迹不断扩大，许多企业开始考虑运营基础设施的可持续方式。VMware 在帮助企业制定可持续计算战略方面迈出了第一步。vSphere 8 引入了绿色指标——可帮助您跟踪工作负载和基础架构操作消耗的电量。这只是帮助客户实现在实现业务目标的同时减少碳足迹的潜力和机会的第一步。
