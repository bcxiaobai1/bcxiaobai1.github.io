二、什么是SSE

1.什么是SSE

了解完HTTP的通信基础，然后说下什么是SSE吧

大家对应HTTP应该不陌生，除了HTTP的通信，应用层还有像websocket，FTP，SMTP，POP3的通信，各有各的适用场景。

今天我们的主角server-send event就是基于HTTP协议的一种流式接口方案，用于服务器向客户端实时发送数据的技术。

它的实现很简单，后端其实只要在http接口响应头里加上

'Content-Type': 'text/event-stream; charset="utf-8"

告知前端接口是流式响应，前端也很简单

1）一种是直接new EventSource("xxx-url");

这种是比较简单的直接拿响应中的event.data即可，这种接口是比较适合无复杂参数，直接通过url传参的流式接口

接口实现：

const http = require('http');

// 创建HTTP服务器
const server = http.createServer((req, res) => {
  // 设置响应头，指定数据类型为text/event-stream
  res.writeHead(200, {
    'Content-Type': 'text/event-stream',
    'Cache-Control': 'no-cache',
    Connection: 'keep-alive',
    'Access-Control-Allow-Origin': '*', // 如果需要允许跨域请求，可以设置该响应头
  });

  // 发送初始化事件
  res.write('event: initn');
  res.write('data: Server connection initializednn');

  // 模拟每秒钟发送一个事件
  const intervalId = setInterval(() => {
    const data = new Date().toISOString();

    // 发送自定义事件和数据
    res.write('event: messagen');
    res.write(`data: ${data}nn`);
  }, 1000);

  // 监听连接断开事件，清除定时器
  req.on('close', () => {
    clearInterval(intervalId);
  });
});

// 启动服务器，监听指定端口
const port = 3000;
server.listen(port, () => {
  console.log(`Server running on port ${port}`);
});

        console.error('Error:', event);
      });

3.SSE与WebSocket的区别

那有的人会问了，那为啥不用websocket呢，我比较熟悉websocket，你这个什么SSE我都没听过，肯定不好用。

那我还是再详细说下websocket的使用场景吧

websocket：它是客户端与服务端或者多客户端与服务端，甚至多客户端之间的互相通信。它更像是群聊一样，谁都能说话，谁都能作为主动方，是双向传输。而SSE它更像是喇叭广播一样，我说什么你们听着就行，只有服务端是主动方能发消息。举例可能不恰当，但就是这么个意思。

4.优点与缺点

SSE的优点：简单易用、低延迟、节省带宽、服务器推送

简单易用前面已经说明了，就是个HTTP请求，只要在响应头和请求头加参数即可

低延迟、节省带宽可以体现在不用像请求大数据接口一样加载慢，耗时长导致服务器压力陡增，容易造成服务器阻塞，假死

服务器推送的好处显而易见，就是能被动的动态获取数据，不用轮询接口拿最新数据了

SSE的缺点：仅支持单向通信、不适用于双向交互，也不能说这是缺点，算是一个特点

3.部署的两种模式

1）cloud模式
docker run -it -e PANDORA_CLOUD=pandora-cloud -e PANDORA_SERVER=0.0.0.0:5500 -p 5500:5500 -d --rm pengzhile/pandora

浏览器打开http://localhost:5500/ 还可以分享给内网的小伙伴内网ip:5500使用

这种模式可以用贴token的形式登录，也可以输入邮箱的形式登录。如果你是谷歌或者微软授权登录的chatGPT，那没辙，只能用token登录的形式了

2）token直接免登陆模式
docker run -it -e PANDORA_ACCESS_TOKEN=你的token -e PANDORA_THREADS=2 -e PANDORA_SERVER=0.0.0.0:6600 -p 6600:6600 -d --rm pengzhile/pandora

和上面一样在浏览器输入http://localhost:6600

可以看到速度很快，不需要开代理，不受网络限制快速回复，调用的都是官方的api

五、总结

在只需要服务端推送数据，又不想造成服务器阻塞，又想让客户端及时得到响应的场景中，就可以用chatGPT这种形式用HTTP的SSE去实现流式加载。

它的特点就是服务端给客户端单向推消息，简单，易用，前后端都不用安装第三方依赖，也不用像websocket单独起个服务，SSE可以直接当一个普通的接口使用，只需要在响应头加'Content-Type': 'text/event-stream; charset="utf-8"'
