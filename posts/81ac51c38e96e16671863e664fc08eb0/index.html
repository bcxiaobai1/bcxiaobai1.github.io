重定向深入了解

一般情况下，每个Unix/Linux 命令运行时都会打开三个文件：

  • 标准输入文件(stdin)：stdin的文件描符，默认从stdin读取数据
  • 标准输出文件(stdout)：stdout 的文件描述符为1，默认向stdout输出数据
  • 标准错误文件(stderr)：stderr的文件描述符为2，向stderr流中写入错误信息

/dev/null 文件
如果希望执行某个命令，但又不希望在屏幕上显示输出结果，那么可以将输出重定向到

/dev/null：

$ command > /dev/null

如果希望屏蔽 stdout 和 stderr，可以这样写：

$ command > /dev/null 2>&1

定时器

[root@master shelldemo]# crontab -e

格式

* * * * * command

可以像使用v i编辑其他任何文件那样修改crontab文件并退出

查看定时任务是否执行了，看日志

[root@master shelldemo]# tail -f /var/log/cron

查看Shell脚本是否执行了，看文件

[root@master shelldemo]# tail -f liangzai.txt

案例

将Shell脚本重定向到liangzai.txt文件中，每秒输出一次，查看日志任务是否执行

1、创建liangzai.txt文本文件

[root@master shelldemo]# touch liangzai.txt

2、创建shell脚本cro.sh

[root@master shelldemo]# vim cro.sh

参考代码：

#!/bin/sh
echo "--------------------------------" >> /usr/local/shelldemo/liangzai.txt
for ((i = 1;i <= 60;i++)) do echo -n "liangzai";echo""; sleep 1;done >> /usr/local/shelldemo/liangzai.txt
echo "--------------------------------" >> /usr/local/shelldemo/liangzai.txt

3、编辑定时器执行Shell脚本cro.sh

* * * * * sh /usr/local/shelldemo/cro.sh

 4、查看定时任务是否执行了，看日志

[root@master shelldemo]# tail -f /var/log/cron

5、查看Shell脚本是否执行了，看文件

[root@master shelldemo]# tail -f liangzai.txt
