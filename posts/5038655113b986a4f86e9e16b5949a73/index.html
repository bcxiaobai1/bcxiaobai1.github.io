云服务器

凭借最高的 SLA 承诺，阿里云 ECS 为全球的云提供商提供了多个可用区。基于大规模分布式计算系统，阿里 ECS 高效且可扩展。阿里巴巴 ECS 包括内部开发的 X-Dragon 计算平台以及 Dragonfly 轻量级虚拟机管理程序。在几分钟内向单个区域中的单个用户提供数十万个 vCPU 的能力是无与伦比的。阿里巴巴的这个智能WEB+平台，帮助您根据业务需求构建稳定安全的Web应用程序。

关系型数据库服务

该关系型数据库服务（RDS）由阿里巴巴公司开发，是最具弹性、高性能、最稳定的数据库服务之一。在提供大部分标准RDS功能的同时，Apsara DB支持OceanBase，PolarDB，MySQL，PostgreSQL，SQL Server，PPAS，MariaDB TX等。此外，阿里巴巴现在为MongoDB提供Apsara DB，进一步涉足支持的数据库平台。

负载均衡

阿里云负载均衡在服务器之间无缝分配流量，从而分配流量负载以保持泛平台的正确功能。它以不会发生单点故障的方式管理 Web 流量，从而消除了 SPOF – 单点故障。阿里云负载均衡支持 TCP 或 UDP（基于源 IP）的第 4 层负载均衡，以及基于 HTTPS 或 HTTP（基于 Cookie）的第 7 层负载均衡，从而扩展了应用程序的服务能力。

对象存储服务

阿里云对象存储服务具有 99.9999999999%（12 个 9）的持久性保证，可提供无缝、加密、极快、可靠的存储备份和归档功能。RESTful API 通过互联网从任何地方轻松访问。借助自动数据复制和恢复功能，您的应用程序正常运行时间将无与伦比。

使用阿里云部署应用程序可以为您带来无与伦比的好处。同时，请考虑以下几点，以充分利用阿里云的功能来部署您的应用程序。

阿里云拥有最先进的安全解决方案，如 WAF（Web 应用程序防火墙）、服务器卫士、防 DDOS 基础版和专业版，以及专用的安全中心。阿里云提供的这些产品可确保在最安全的环境中托管应用程序，因为大多数应用程序漏洞问题都已由这些产品解决。

使用 Server Guard，您可以通过轻松定义防火墙规则和例外来限制或控制流量流。在访问Web服务器集群时，预定义的防火墙规则仅允许访问端口80和443（http/https请求），并且为了进一步保护应用程序服务器，安全组规则仅允许通过Web服务器进行访问。

对于数据库服务器，预定义的安全组允许访问应用程序数据，并且仅允许通过端口 22 （SSH） 直接管理主机通过应用程序层进行访问。您可以进一步配置此组，以允许列入白名单的 IP 地址通过防火墙进行访问。

阿里云定义可用区，并确保为单个区域分配至少两个数据中心。在弹性云服务器中，不同的可用区由这些可用区分隔，但阿里云提供了一个非常简单的模型来跨可用区部署您的应用程序，以实现更高的可用性和弹性。

为云构建应用程序需要一个本身是动态的平台，并且了解可用性和可靠性的概念。阿里巴巴ECS正是这样做的。在部署应用程序时，阿里云允许用户选择资源的位置，并提供弹性 IP 地址。这解决了部署应用程序时非常常见的确定性问题。阿里云的简单应用服务器管理并分离从前端到后端的信息流，反之亦然，以保持安全可靠的通信形式。

让我们看一个关于失败的场景。在这种情况下，您的所有资源都可能变得不可用。现在，当分配新的虚拟服务器时，将向其分配哪个区域和 IP 更有可能是猜测工作。在这里，阿里云进来了，并为用户提供了弹性 IP 和资源位置首选项，从而克服了这个问题。即使您在新的 ECS 实例中进行了改动，您也可以简单地将弹性公网IP映射到新的 ECS 实例。这说明了可扩展和抗故障的应用程序构建和托管。
