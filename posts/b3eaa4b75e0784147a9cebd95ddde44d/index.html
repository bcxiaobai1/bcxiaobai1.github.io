Android模拟器 7.1 （64） 以上版本安装xposed框架

在xposted之前的版本里 xposed只支持 android 4.4以下的版本，因此当时 逍遥模拟器的android版本就是4.4，而雷电模拟器、蓝叠模拟器、夜神模拟器等安装后的android版本都在5.0以上，因此流传着xposed配合逍遥。 看了很多文章 以及逆向的视频教程 都这么说。 接下来我们就尝试一下整个过程，以及遇到的各种问题  和 解决方案。

一、安装雷电模拟器

常用的模拟器我都安装完毕，这里我们先体验一下逍遥模拟器xposed框架

 

 

二、按照网上流传的常规教程先下载 xposed框架， 百度搜索一堆，随便找一个（担心文件丢失，放在网盘里了 大家自行下载）

 点击框架进行安装

 

 

xposed安装失败，根本原因还是如网上资料所讲 xposed不支持 64位的  和 Android 5以上的版本。

于是尝试去找一下xeposed 更高的版本 x86_64 xposed framework， 此种方式安装就不是那么傻瓜式的安装了。接下来进行安装过程。

三、下载  你当前模拟器支持的xposed框架版本

Index of /framework/https://dl-xda.xposed.info/framework/

下载解压后 记得将文件重命名为 xposed

 下载 脚本文件

x86_64处理器的从这里下载：script.txt

x86处理器的从这里下载：script.txt

 四、安装

1、将xposed文件导入模拟器/system目录下，可通过模拟器的共享文件夹，设置好共享文件夹，现将xposed放入PC共享文件夹中如图一，再在模拟器端使用文件管理器将xposed移动到/system目录下

注意: 不是所有模拟器都彻底root了的， 有些模拟器无法通过文件管理 访问到 / 根目录 和 /system 等一些系统文件。（即便你在设置里 root  打勾）

这里我们采用 adb 进行安装，先确定adb已经连接到模拟器

C:UsersAdministrator>adb devices
List of devices attached
emulator-5554   device

push 到手机的 /system/ 目录下（可能由于权限问题无法直接push到/system）

C:UsersAdministrator>adb push E:xposed  /system/
adb: error: failed to copy 'E:xposedscript.sh' to '/system/xposed/script.sh': remote secure_mkdirs failed: Read-only file system
adb: error: failed to read copy response

C:UsersAdministrator>

Tips: 推荐先把文件 push 到 sdcard 目录，在使用 mv 移动到 system 目录

C:UsersAdministrator>adb push E:xposed  /sdcard/Pictures/
E:xposed: 17 files pushed, 0 skipped. 7.1 MB/s (28455692 bytes in 3.829s)

C:UsersAdministrator>
 

 

2、进入模拟器终端

C:UsersAdministrator>adb shell
aosp:/ #   

3、提权 并将 xposed文件 移动到 /system目录下

aosp:/ # cd /sdcard/Pictures

aosp:/sdcard/Pictures # mv xposed/ /system/
mv: /system//xposed: Read-only file system

进入模拟器中断后，有些模拟器的文件夹路径会不一样，比如逍遥会提示 /sdcard/Picutres/xposed 不存在，因为进入模拟器终端后系统文件路径与进入终端之前的adb命令下路径不同，应该是 storage/emulated/0/Pictures

当前问题是 无法操作 /system 这种系统文件，因为文件是只读，重新挂载目录即可

1|aosp:/sdcard/Pictures # mount -o rw,remount /system

 再次尝试，执行成功

aosp:/sdcard/Pictures # mv xposed/ /system/

4、进入xposed目录 ，给脚本赋予执行权限，执行script脚本

aosp:/sdcard/Pictures # cd /system/xposed/
aosp:/system/xposed # ls
script.sh  system
aosp:/system/xposed # chmod 777 script.sh
aosp:/system/xposed # ./script.sh
