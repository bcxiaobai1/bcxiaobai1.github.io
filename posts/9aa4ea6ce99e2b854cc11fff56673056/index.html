编程小白

一文解决在虚拟机Kubernetes集群初始化

环境：CentOS7.9.2009
配置要求虚拟机设置至少二核cpu，内存1g

一、修改主机名

ip a | grep 192 或 ifconfig | grep 192 查询ip地址
vim /etc/hostname
修改为 node1
vim /etc/hosts
添加 192.168.119.131 node1
192.168.119.132 node2 等

二、禁用selinux

手动关闭 setenforce 0
禁用 vim /etc/selinux/config，修改SELINUX值为disabled（重启生效，可以getenforce验证）

三、禁用swap分区

vim /etc/fstab，注释掉swap那行（重启生效，可以用free -m验证）

四、修改内核参数

vim /etc/sysctl.d/kubernetes.conf，添加如下配置(这是个新文件)
net.bridge.bridge-call-ip6tables = 1
net.bridge.bridge-call-iptables = 1
net.ipv4.ip_forward = 1
执行以下命令：
sysctl -p
modprobe br_netfilter
lsmod | grep br_netfilter
修改linux的内核参数，添加网桥过滤和地址转发功能

五、ipvs

①安装ipset和ipvsadm
yum install -y ipset ipvsadm
②配置
modprobe – ip_vs
modprobe – ip_vs_rr
modprobe – ip_vs_wrr
modprobe – ip_vs_sh
modprobe – nf-conntrack_ipv4
先跑一遍来手动设置，然后把上面的写入vim /etc/sysconfig/modules/ipvs.modules
查看是否配置成功
lsmod | grep -e ip_vs -e nf_conntrack_ipv4

六、禁用iptables和firewalld

systemctl stop firewalld
systemctl disable firewalld
这是为了不让系统规则和k8s、docker规则混淆。
systemctl stop iptables
systemctl disable iptables
然后时间同步一下（在虚拟机上同步不同步无所谓）
systemctl start chrondy
systemctl enable chrondy
以上操作完成后，reboot。然后通过free -m，getenforce、ping node2等命令检验。

七、事先把containerd安装好

①在github的containerd里面点击页面右侧的releases，然后在windows上下载containerd-1.6.2-linux-amd64.tar.gz。
②然后在通过在windows上面scp的方式拷贝到linux中，注意修改地址
scp C:UsersadministatorDownloadscontainerd-1.6.2-linux-amd64.tar.gz root@192.168.119.131:/home/reo/Desktop
③tar xvf con…之后cd到bin目录下，mv * /bin把里面的程序移动到bin目录

八、安装Kubernetes

vim /etc/yum.repos.d/kubermetes.repo
在其中添加，给yum配置Kubernetes镜像源
[kubernetes]
name=Kubernetes
baseurl=http://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-x86_64
enabled=1
gpgcheck=0
repo_gpgcheck=0
gpgkey=http://mirrors.aliyun.com/kubernetes/yum/doc/yum-key.gpg
http://mirrors.aliyun.com/kubernetes/yum/doc/rpm-package-key.gpg
然后yum install，先查看一下版本有哪些，然后直接装最新版本
yum list kubelet --showduplicates
yum install kubeadm-1.23.5-0 kubelet-1.23.5-0 kubectl-1.23.5-0 -y
（黑马的教程说是还需要修改vim /etc/sysconfig/kubelet，在里面添加，但是发现不做这步也没事）
KUBELET_CGROUP_ARGS="–cgroup-driver=systemd"
KUBE_PROXY_MODE=“ipvs”

九、安装Docker

首先搞镜像源
wget https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo -O /etc/yum.repos.d/docker-ce.repo
查看docker版本
yum list docker-ce --showduplicates
（可以这样安装特定版本的docker
yum install --setopt=obsoletes=0 docker-ce-18.06.3.ce-3.el7 -y）
然而我们并不需要一个特定版本的docker，反正k8s都不会用到它，所以可以直接
yum install docker-ce -y
添加一个配置文件，给docker配置镜像源
mkdir /etc/docker
cat < /etc/docker/daemon.json
{
“exec-opts” : [“native.cgroupdriver=systemd”],
“registry-mirrors” : [“https://kn0t2bca.mirror.aliyuncs.com”]
}
EOF
启动docker服务并设置开机启动
systemctl restart docker
systemctl enable docker

十、使用docke拉取k8s镜像组件（一 ~ 十的操作在所有节点执行）

可以通过让kubeadm的init自动超时，来看看本地版本的k8s需要哪些版本的镜像组件，在我们使用的v1.23.5版本中，需要的组件版本如下，首先定义一个images集合，然后运行一个SHELL脚本：
images=(
kube-apiserver:v1.23.5
kube-controller-manager:v1.23.5
kube-scheduler:v1.23.5
kube-proxy:v1.23.5
pause:3.6
etcd:3.5.1-0
coredns:v1.8.6
)
for imageName in i m a g e s [ @ ] ; d o d o c k e r p u l l r e g i s t r y . c n − h a n g z h o u . a l i y u n c s . c o m / g o o g l e c o n t a i n e r s / {images[@]}; do docker pull registry.cn-hangzhou.aliyuncs.com/google_containers/ images[@];dodockerpullregistry.cn−hangzhou.aliyuncs.com/googlec​ontainers/imageName
docker tag registry.cn-hangzhou.aliyuncs.com/google_containers/ i m a g e N a m e k 8 s . g c r . i o / imageName k8s.gcr.io/ imageNamek8s.gcr.io/imageName
docker rmi registry.cn-hangzhou.aliyuncs.com/google_containers/$imageName
done
下载完成之后可以用docker images查看，会发现拉取的国内镜像源都被成功tagged为k8s标准镜像名了然而，通过init超时的警告可以发现其中coredns地址和其他的有点区别），因此需给coredns改名。
docker tag k8s.gcr.io/coredns:v1.8.6 k8s.gcr.io/coredns/coredns:v1.8.6
docker rmi k8s.gcr.io/coredns:v1.8.6
然后启动kubelet服务
systemctl start kubelet
systemctl enable kubelet
至此组件镜像准备工作完成。

十一、kubeadm init（在master节点执行）

然后我们就可以进行kubeadm init了，注意修改版本号和ip地址
kubeadm init --kubernetes-version=v1.23.5 --pod-network-cidr=10.24.0.0/16 --service-cidr=10.96.0.0/12
–apiserver-advertise-address=192.168.119.131
如果init成功，那么系统就会提示帮助信息，按照他的步骤就是：
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown ( i d − u ) : (id -u): (id−u):(id -g) $HOME/.kube/config
然后如果有多个节点的话，就按照帮助信息，在每个node上运行kubeadm join的那一串帮助信息
创建完成后可以通过kubectl get nodes查看节点信息，可以注意到STATUS状态显示NotReady，这是因为还没有配置网络，将在接下来配置。
这时候输入kubectl version不再报错。

十二、join node（在每个node节点以root执行）

参考init时返回的关于join的信息，如果忘了，可以通过以下命令查询
kubeadm token create --print-join-command

十三、配置网桥（在master节点执行）

把配置地域文件夹下的kube-flannel.yml scp到linux，然后
kubectl apply -f kube-flannel.yml
可以通过kubectl get nodes检验，发现节点状态变为了Ready

kube-flannel.yml

---
apiVersion: policy/v1beta1
kind: PodSecurityPolicy
metadata:
  name: psp.flannel.unprivileged
  annotations:
    seccomp.security.alpha.kubernetes.io/allowedProfileNames: docker/default
    seccomp.security.alpha.kubernetes.io/defaultProfileName: docker/default
    apparmor.security.beta.kubernetes.io/allowedProfileNames: runtime/default
    apparmor.security.beta.kubernetes.io/defaultProfileName: runtime/default
spec:
  privileged: false
  volumes:
    - configMap
    - secret
    - emptyDir
    - hostPath
  allowedHostPaths:
    - pathPrefix: "/etc/cni/net.d"
    - pathPrefix: "/etc/kube-flannel"
    - pathPrefix: "/run/flannel"
  readOnlyRootFilesystem: false
  # Users and groups
  runAsUser:
    rule: RunAsAny
  supplementalGroups:
    rule: RunAsAny
  fsGroup:
    rule: RunAsAny
  # Privilege Escalation
  allowPrivilegeEscalation: false
  defaultAllowPrivilegeEscalation: false
  # Capabilities
  allowedCapabilities: ['NET_ADMIN']
  defaultAddCapabilities: []
  requiredDropCapabilities: []
  # Host namespaces
  hostPID: false
  hostIPC: false
  hostNetwork: true
  hostPorts:
  - min: 0
    max: 65535
  # SELinux
  seLinux:
    # SELinux is unused in CaaSP
    rule: 'RunAsAny'
---
kind: ClusterRole
apiVersion: rbac.authorization.k8s.io/v1beta1
metadata:
  name: flannel
rules:
  - apiGroups: ['extensions']
    resources: ['podsecuritypolicies']
    verbs: ['use']
    resourceNames: ['psp.flannel.unprivileged']
  - apiGroups:
      - ""
    resources:
      - pods
    verbs:
      - get
  - apiGroups:
      - ""
    resources:
      - nodes
    verbs:
      - list
      - watch
  - apiGroups:
      - ""
    resources:
      - nodes/status
    verbs:
      - patch
---
kind: ClusterRoleBinding
apiVersion: rbac.authorization.k8s.io/v1beta1
metadata:
  name: flannel
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: flannel
subjects:
- kind: ServiceAccount
  name: flannel
  namespace: kube-system
---
apiVersion: v1
kind: ServiceAccount
metadata:
  name: flannel
  namespace: kube-system
---
kind: ConfigMap
apiVersion: v1
metadata:
  name: kube-flannel-cfg
  namespace: kube-system
  labels:
    tier: node
    app: flannel
data:
  cni-conf.json: |
    {
      "name": "cbr0",
      "cniVersion": "0.3.1",
      "plugins": [
        {
          "type": "flannel",
          "delegate": {
            "hairpinMode": true,
            "isDefaultGateway": true
          }
        },
        {
          "type": "portmap",
          "capabilities": {
            "portMappings": true
          }
        }
      ]
    }
  net-conf.json: |
    {
      "Network": "10.244.0.0/16",
      "Backend": {
        "Type": "vxlan"
      }
    }
---

                      - amd64

      serviceAccountName: flannel

        image: quay.io/coreos/flannel:v0.11.0-amd64
        command:
        - cp
        args:
        - -f
        - /etc/kube-flannel/cni-conf.json
        - /etc/cni/net.d/10-flannel.conflist

      - name: kube-flannel
